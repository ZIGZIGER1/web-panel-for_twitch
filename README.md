# Web panel by ZigZiger

Локальная панель для стрима и OBS.  
Она запускает web-интерфейс в браузере, отдает прозрачную сцену для `Browser Source`, показывает персонажа, управляет кастомным Twitch-чатом и живет в системном трее.

## Что это

Проект нужен, если ты хочешь:
- вывести персонажа в OBS без возни с отдельным тяжелым софтом;
- получить локальный прозрачный URL для сцены;
- отдельно вывести чат как свой overlay;
- запускать панель без черного консольного окна.

## Что умеет

- Web-панель управления сценой, чатом и пресетами.
- Прозрачная сцена для OBS через локальный `Browser Source`.
- Персонаж с тремя кадрами: `1.png`, `2.png`, `3.png`.
- Twitch-чат с отдельным overlay и режимом появления только при сообщениях.
- Запуск через трей: приложение открывает панель и остается в фоне.
- Сборка в `exe` и portable-архив для раздачи другим людям.

## Как запустить

### Вариант 1. Из исходников

```powershell
py -m pip install -r requirements.txt
py mopsyan_sysan.py
```

Или просто запусти `start_mopsyan_sysan.bat`.

### Вариант 2. Готовый exe

Готовые файлы после сборки:

- `dist/Web_panel_by_ZigZiger.exe`
- `release/Web_panel_by_ZigZiger_portable.zip`

## Как добавить в OBS

### Сцена

1. Запусти панель.
2. Открой локальный адрес, который она показала в браузере.
3. В панели найди адрес сцены.
4. В OBS добавь новый `Browser Source`.
5. Вставь туда URL сцены.

### Чат

1. В панели открой раздел чата.
2. Укажи Twitch-канал или ссылку на чат.
3. Возьми chat URL из панели.
4. В OBS добавь второй `Browser Source`.
5. Вставь туда адрес чата.

## Как настроить микрофон

1. В панели открой блок `Микрофон и чувствительность`.
2. Нажми `Обновить`, чтобы получить список доступных устройств.
3. Выбери нужный микрофон в поле `Устройство`.
4. Нажми `Старт`.
5. Скажи что-нибудь в микрофон и посмотри на `Текущий уровень`.
6. Подстрой `Порог речи` так, чтобы персонаж не дергался от фонового шума, но нормально реагировал на голос.

Если микрофон не появился в списке, проверь его в настройках Windows и снова нажми `Обновить`.

## Что вставлять в поле чата

В поле `Канал или ссылка` можно вставлять:

- `@имя_канала`
- `twitch.tv/имя_канала`
- полную ссылку на канал Twitch
- ссылку вида `https://www.twitch.tv/popout/имя_канала/chat`
- ссылку вида `https://www.twitch.tv/embed/имя_канала/chat`

Если нужен более стабильный режим, в тех же настройках можно дополнительно указать:

- `Логин Twitch`
- `OAuth token`

## FAQ

### Почему после запуска открывается браузер?

Потому что основная панель у проекта web-овая. Это не ошибка и не реклама: приложение поднимает локальный сервер на твоем ПК и открывает интерфейс в браузере.

### Почему программа не висит как обычное окно?

Потому что основной режим запуска сделан через системный трей. Так панель не оставляет черное консольное окно и не мешает во время стрима.

### Как закрыть программу?

Нажми правой кнопкой по иконке в трее и выбери `Закрыть`.

### Где менять картинки персонажа?

Замени `1.png`, `2.png` и `3.png` в папке проекта или укажи свои изображения в панели.

### Почему адрес сцены не всегда ровно `127.0.0.1:8765`?

По умолчанию приложение старается занять этот порт, но если он уже используется, оно поднимается на ближайшем свободном. Поэтому правильный адрес лучше копировать прямо из панели.

### Где хранятся настройки?

В `%LOCALAPPDATA%\\mopsyan_sysan`.

## Сборка exe

```powershell
py build_exe.py
```

Если `exe` уже собран и нужно только обновить portable-пакет:

```powershell
py build_exe.py --package-only
```

## Что есть в репозитории

- `web/` — интерфейс панели
- `assets/ui/` — графика интерфейса
- `web_runtime.py` — локальный runtime и сервер
- `scene_renderer.py` — рендер сцены
- `chat_overlay.py` — логика чата и overlay
- `mopsyan_tray.py` — запуск и управление через трей
- `build_exe.py` — сборка `exe` и portable-пакета

В репозиторий специально не включаются `build/`, `dist/`, `release/` и временные локальные файлы.
